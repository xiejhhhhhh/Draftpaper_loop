# Forensic Fingerprinting

Draftpaper-loop uses visible, auditable, deterministic schema and behavior
contracts to preserve compatibility and provenance across releases.

The project does not use hidden executable payloads, invisible watermarks,
telemetry, device fingerprinting, remote license checks, destructive
anti-removal behavior, or malware-like tricks.

Maintainers may compare release artifacts, schema contracts, generated
manifests, project passports, citation-evidence layouts, and deterministic
edge-case outputs to evaluate compatibility and provenance of derivative
systems.

## Public Layer

The public layer includes:

- source-available non-commercial license terms;
- commercial-use and compliance documents;
- trademark and associated-project-term notices;
- documented DPL schema family names;
- visible generated metadata;
- deterministic identifier formats such as claim and evidence IDs;
- compatibility tests for schema and project metadata.

## Private Layer

Private forensic cases, golden outputs, clone-comparison scripts, and exact
similarity thresholds are intentionally not part of the public repository. This
keeps public documentation useful for legitimate users while avoiding release of
private review material.

Private material should stay outside the public repository and should never be
required for normal user execution.
