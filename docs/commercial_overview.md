# Draftpaper-loop Commercial Overview

Draftpaper-loop is a local-first research workflow for teams that need traceable scientific execution, manuscript construction and controlled author revision. It is source-available for non-commercial use under `LicenseRef-Draftpaper-NonCommercial`; commercial use requires written authorization.

## Current Product Boundary

- Local CLI and optional local stdio MCP, with projects and private data remaining on the user's machine or approved infrastructure.
- Evidence-first research planning, discipline-plugin binding, run manifests, semantic figure contracts, manuscript writing, citation audit, independent review and release-hash confirmation.
- Explicit minimal, plotting, fulltext and MCP install profiles.
- Command-level risk, input/output, write-set, confirmation and recovery contracts.
- Wheel, schema, provenance, secret-scan and cross-discipline release verification.

Draftpaper-loop does **not currently provide a public hosted API**. Multi-tenant authentication, tenant isolation, controlled outbound networking and hosted data-governance guarantees are outside the current release boundary. Local command allowlists and project write-set protection must not be represented as a complete cloud sandbox.

## Commercial Evaluation

A commercial evaluation should define deployment location, user count, support scope, required discipline plugins, private connector responsibilities, security review, data-retention policy and distribution method. Mock, fixture-only and contract-only plugins demonstrate interfaces only; live scientific capability must be validated against the customer's actual environment.

Contact and maintenance information are available on the project support page. Authorization does not transfer ownership of third-party components; their notices and license records remain applicable.
