# Draftpaper-loop 商业能力概览

Draftpaper-loop 是面向科研团队的本地优先工作流，用于组织可追溯科研执行、论文构建和受控作者修订。项目依据 `LicenseRef-Draftpaper-NonCommercial` 允许非商业使用；商业使用需要书面授权。

## 当前产品边界

- 本地 CLI 和可选本地 stdio MCP，论文项目和私有数据保留在用户设备或批准的基础设施中。
- Evidence-first 研究计划、学科插件绑定、run manifest、语义图表合同、论文写作、引用核查、独立审稿和 release hash 确认。
- 显式 minimal、plotting、fulltext 和 MCP 安装档位。
- 命令级风险、输入输出、write-set、人工确认和恢复合同。
- Wheel、schema、第三方来源、密钥扫描和跨学科发布验证。

Draftpaper-loop **当前不提供公网托管 API**。多租户鉴权、租户隔离、受控出站网络和托管数据治理不属于当前版本能力。本地命令 allowlist 和项目 write-set 保护不能被宣传为完整云端 sandbox。

## 商业评估

商业评估应明确部署位置、用户数量、支持范围、所需学科插件、私有 connector 责任、安全审查、数据保留策略和分发方式。Mock、fixture-only 和 contract-only 插件只能展示接口；真实科研能力必须在客户实际环境中验证。

联系方式和维护信息见项目支持页面。商业授权不会转移第三方组件的所有权，其 notices 和许可证记录仍然有效。
